计图挑战赛赛道一热身赛
赛题介绍
本赛道将使用 JittorGeometric 图神经网络库，在基于 Cora 的引文网络数据集上训练 GCN（Graph Convolutional Network）模型，完成节点分类任务。

赛题内容
本赛题将提供一个基于 Cora 的引文网络数据集（data/cora.pkl，已包含在发布包中）。参赛选手需要基于 JittorGeometric 库，使用 GCN 模型完成节点分类任务，对测试集中的节点预测其所属类别。

任务说明
输入：引文网络图数据（data/cora.pkl），已包含在发布包中
输出：提交格式为测试集节点的预测分类，保存为 JSON 文件，格式为字典，key 为节点编号（int），value 为预测类别（int）
注意：数据集中测试集节点的标签为 -1（已隐藏），选手需要通过训练模型来预测
数据集简介
本赛题使用的数据集基于经典的 Cora 引文网络数据集：

节点数：2708（每个节点代表一篇论文）
边数：5429（边代表引用关系）
特征维度：1433（词袋特征）
类别数：7
数据集以 pickle 格式提供，包含以下字段：

x：节点特征矩阵（numpy array, shape: 2708×1433）
y：节点标签（numpy array, shape: 2708）。训练集/验证集标签完整，测试集标签为 -1
edge_index：边列表（numpy array, shape: 2×num_edges）
train_mask、val_mask、test_mask：训练集、验证集、测试集掩码
num_classes：类别数
num_features：特征维度
数据集与示例代码
本赛题提供数据集与示例代码框架，提供数据加载、模型定义、训练步骤等功能。

选手可以基于示例代码填充注释为 TODO 的部分完成该赛题。

评测方式
评测程序会加载选手提交的预测结果（result.json），与测试集的真实标签进行对比，计算分类准确率（Accuracy）。

测试集准确率 ≥ 0.70 即为通过。
提交结果格式说明
将模型代码与预测结果打包为 zip 提交，目录结构和文件名按照以下方式组织：

-- result.zip
  -- gcn.py        // 模型代码 + 训练代码
  -- result.json   // 测试集预测结果
其中 result.json 的格式示例：

{
    "1708": 0,
    "1709": 1,
    "1710": 3,
    ...
}
key 为测试集节点的编号（字符串），value 为预测的类别编号（整数，0-6）。
