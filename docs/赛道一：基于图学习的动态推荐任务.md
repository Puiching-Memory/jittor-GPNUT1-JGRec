## 赛题介绍

近年来，随着推荐系统、社交网络、交易网络等图数据的不断丰富，时序图学习成为图机器学习领域的重要研究方向。与静态图不同，时序图中的节点关系会随时间不断演化，模型不仅需要理解图结构本身，还需要捕捉交互行为背后的时间规律与序列模式。

本赛题聚焦时序图中的未来链接预测任务。给定用户、物品、作者、网页等实体在历史上的交互序列，模型需要预测未来最有可能发生的新连接。具体来说，赛题会提供一系列时序交互三元组（source, destination, time），参赛者需要结合序列推荐或时序图学习方法进行建模，学习用户、物品或节点之间的时序关联模式，并在给定 time 和 source 的情况下，预测其未来可能交互的 destination。

例如在社交网络情境下，已知用户 A、B、C、D 的历史交互记录为：

- 时刻 0：用户 A 发起与 B 的交互（A, B, 0）
- 时刻 1：用户 B 发起与 C 的交互（B, C, 1）

在上述时序交互三元组中，source 和 destination 均对应于社交网络中的用户节点。模型需要预测用户节点 A 于时刻 2 的可能交互，给出与候选节点产生交互的概率，例如 `[B, C, D]`：`(0.40, 0.50, 0.10)`，代表预测用户节点 A 于时刻 2 分别与用户节点 B、C、D 发起交互的概率。

本赛道的任务可建模为时序图节点重排序任务，我们为参赛者提供了基于 JittorGeometric 库实现的代码作为 Baseline，使用方法详见仓库的 README 说明。

## 赛题意义与挑战

该任务广泛存在于商品推荐、好友推荐、论文引用预测、网页跳转预测等场景中，具有重要的研究价值与应用意义。近年来研究表明，现有不少时序图数据集存在重复边比例较高的问题，模型往往更容易依赖对历史连接的记忆，而不是学习更加真实且更具挑战性的序列动态规律。因此，如何提升模型对未来新连接的建模能力与泛化能力，仍然是时序图学习中的关键挑战。

## 赛题内容

### 数据下载

A 榜训练数据和测试数据集。

A 榜提供难度适中的数据集，规律易于挖掘。B 榜提供难度更大的数据集，节点数百万级，交互千万级，需要复杂建模并控制计算效率。选手在 A/B 榜提供的算法应当基本一致，能够兼顾各种数据量级的推荐。

简单和复杂数据集均包含二部图和非二部图，二部图的 src 和 dst 分属不同集合，非二部图的 src 和 dst 在同一集合。

### 数据说明

训练集每行包含两个节点编号 `u`、`v` 和时间戳 `t`，含义为源节点 `u` 和目标节点 `v` 在时间 `t` 有交互。

```text
u1, v1, t1
u2, v2, t2
u3, v3, t3
u4, v4, t4
u5, v5, t5
```

测试集每行包含一个节点编号 `u`、时间戳 `t`、100 个候选目标节点，参赛选手需返回在时间戳 `t` 时，源节点 `u` 和 100 个候选目标节点的交互概率分布。

```text
u1, t1, v1, ..., v100
```

## 评测指标

赛题分为 A、B 两个榜单。A 榜结束后将按排名筛选、审核让若干支队伍进入 B 榜。

A 榜评分公式：采用各场景预测结果的 MRR（Mean Reciprocal Rank，平均倒数排名）均值之和。如果第一个相关的结果出现在第 `k` 位，那么这个查询的得分（RR）就是 `1/k`。每个场景的 MRR 均值定义为：

```text
MRR = (1 / |Q|) * Σ(1 / rank_i)
```

B 榜评分公式：各场景预测结果的 MRR 之和 * 65% + 35% 答辩分数。

## 提交格式说明

A、B 榜每个数据集提交一个 csv 文件，所有文件打包为 `result.zip`。每个 csv 文件命名与数据集场景一致，每行包含源节点和 100 个候选节点发生交互的概率，每个概率之间以英文半角逗号（`,`）分隔，每个概率值需在 `[0, 1]` 之间，保留 8 位小数。

