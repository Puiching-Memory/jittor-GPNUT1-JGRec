# 数据契约

## 输入目录

程序以数据集目录为处理单位：

```text
data/<dataset_name>/
├── train.csv
└── test.csv
```

`<dataset_name>` 会用于输出文件名：`result/<run_id>/csv/<dataset_name>.csv`。

## 训练集

训练集必须包含以下列：

| 列名   | 类型 | 说明         |
| ------ | ---- | ------------ |
| `src`  | int  | 源节点编号   |
| `dst`  | int  | 目标节点编号 |
| `time` | int  | 交互时间戳   |

允许存在额外列，例如 `split`。当前 MVP 会忽略额外列。

示例：

```csv
src,dst,time
2,3,0
5,5,699798
2,3,764356
```

## 测试集

测试集必须包含：

| 列            | 类型 | 说明               |
| ------------- | ---- | ------------------ |
| `src`         | int  | 源节点编号         |
| `time`        | int  | 查询时间戳         |
| `c1`...`c100` | int  | 100 个候选目标节点 |

示例：

```csv
src,time,c1,c2,...,c100
8512,129595642,25029,9278,...,17224
```

当前代码严格要求候选列数量为 100。

## 输出 CSV

每个数据集生成一个同名 CSV：

```text
result/<run_id>/csv/<dataset_name>.csv
```

输出约束：

- 无表头。
- 行数等于对应 `test.csv` 的数据行数。
- 每行 100 个概率，对应测试集中 `c1` 到 `c100` 的顺序。
- 每个概率在 `[0, 1]`。
- 每个概率保留 8 位小数。

示例：

```csv
0.00998798,0.01091745,...,0.00899571
0.00880761,0.00699108,...,0.00935113
```

## 提交压缩包

最终提交文件：

```text
result/<run_id>/result.zip
├── dataset1.csv
├── dataset2.csv
└── ...
```

压缩包文件名固定为 `result.zip`，根目录直接包含 CSV 文件，不包含 `result/` 或 `csv/` 目录层级。

## 校验规则

`jgrec.submission.validate_submission_file()` 执行以下校验：

- 列数必须为 100。
- 概率范围必须在 `[0, 1]`。
- 全量运行时，输出行数必须匹配测试行数。

`--limit-rows` 冒烟测试会跳过行数等于完整测试集的校验。
