# 当前数据画像

本文记录 2026-05-29 对本地 `data/` 的统计和特征分析。结论用于指导模型选择、负采样协议和后续特征工程。

原始结果：

```text
result/benchmarks/current_data_profile.json
result/benchmarks/current_data_deep_profile.json
```

复现深度画像：

```bash
uv run python scripts/analyze_data_profile.py --output result/benchmarks/current_data_deep_profile.json
```

## 统计协议

- 数据源：本地 `data/dataset1` 和 `data/dataset2`。
- 全量画像：读取完整 `train.csv` 和 `test.csv`。
- holdout 画像：使用 `train.csv` 前 85% 作为历史，后 15% 作为未来事件。
- 测试候选画像：读取完整 `test.csv`，统计每行 100 个候选在训练历史中的覆盖情况。
- 特征代理验证：在 holdout 正样本上构造两类候选集：
  - `random`：99 个随机训练 `dst` 负样本。
  - `test_like`：优先从同源测试候选和全局测试候选分布采样负样本。
- 抽样规模：单特征/分段代理 3000 条正样本，图信号代理 1000 条正样本，融合消融 2000/3000 条训练/验证正样本。

注意：所有代理验证仍不是线上指标。本文主要看 AUC、tie-MRR、strict Hit@1、非零率和分段差异。

## 统计融合消融

使用 `test_like` 候选协议训练 Logistic Regression 融合现有 8 个统计特征，并做 permutation importance。

| 数据集     |     AP | tie-MRR | strict Hit@1 | median tie-rank |
| ---------- | -----: | ------: | -----------: | --------------: |
| `dataset1` | 0.3966 |  0.5456 |       0.4643 |             2.0 |
| `dataset2` | 0.1316 |  0.3694 |       0.2217 |             5.0 |

Permutation 按 tie-MRR drop 排序：

| 数据集     | 关键特征         | tie-MRR drop | AP drop |
| ---------- | ---------------- | -----------: | ------: |
| `dataset1` | `pair_recency`   |       0.4670 |  0.3825 |
| `dataset1` | `dst_popularity` |       0.0182 |  0.0103 |
| `dataset1` | `dst_recency`    |       0.0049 |  0.0041 |
| `dataset2` | `dst_popularity` |       0.3284 |  0.1231 |
| `dataset2` | `src_recency`    |       0.0756 |  0.0479 |
| `dataset2` | `src_activity`   |       0.0168 |  0.0020 |

含义：

- `dataset1` 当前融合几乎由 pair recency 决定，其他特征只做兜底。
- `dataset2` 当前融合主要依赖 dst popularity，source recency/activity 有补充价值；pair 类特征不应占主导。

## 建模建议

1. `dataset1` 保持强统计记忆主干：pair 频次、pair 近因、repeat rate、recent window 必须稳定进入融合。
2. `dataset1` 补短板：对“已知节点新 pair”和“未见 `dst`”增加同构图反向边、共现、source-source/dst-dst 相似特征。
3. `dataset2` 把主任务视为已知节点新链接预测，而不是重复边预测；pair 记忆和 CRAFT 近期邻居匹配不作为主线。
4. `dataset2` 优先做目标热度/近因、source profile、二跳共现、source-dst 图泛化和候选分布校准。
5. `dataset2` 对未见 `dst` 必须设计兜底策略：当前统计特征在该分段 AUC 接近 0.5，纯 ID embedding 也无法覆盖。
6. 后续本地验证必须加入 `test_like` 候选负采样；随机负采样只能做初筛，不能单独决定模型进入默认链路。
